Visualizing your code

The staff has written a graphical interface to the project to allow you to visualize how your algorithms go about solving puzzles. The GUI allows you to view initial configurations, see your algorithm run (at chosen speeds) on an initial configuration, and move manually through the moves generated by your code. We recommend that you use the GUI to explore different search algorithms and the optimality of the move lists that they produce. Note that faster algorithms may not necessarily be optimal. Your aim in this project is the speed of your algorithm.

Using the GUI

The GUI requires two files to simulate an algorithm: the initial block configuration file, and the list of moves output by your code. Alternatively, if you just want to view an initial tray configuration, click "cancel" when it asks you for a moves file. To get started using the GUI:

  • Move to the directory where you plan to conduct all your testing. This directory should have all your class files, as well as copied versions of any tests you plan on running.
  • Create output files of the moves that your solver produces by entering: 
    java Solver inputfile goalfile > outputfile
    For example: 
    java Solver 1x1 1x1.goal > 1x1.out
  • Then copy proj3GUI.jar to this directory. (A "jar" file can be thought of as a library of java classes that are ready to be used.) 
    cp ~cs61bl/code/proj3/gui/proj3GUI.jar .
    Note the period at the end of the cp.
  • Run the jar and follow its instructions. 
    java -jar proj3GUI.jar

Features

  • You can pause, play and step through execution of moves.
  • You can adjust the delay between the animation of each move by dragging the slider.
  • A progress bar indicates the percentage of moves that have already been executed.
  • The GUI adaptively sets the default delay based upon the number of moves that you have to execute.

Keyboard Shortcuts

  • Pause: Alt + p
  • Play: Alt + Enter
  • Next move: Alt + n
  • Previous move: Alt + r
